# Available timezone strings

`configTzTime()` on ESP32 uses POSIX `TZ` strings. These are not IANA timezone names such as `America/Los_Angeles`; they are compact rules that describe the local UTC offset and, optionally, daylight-saving transitions.

Use these strings as the timezone value in Wi-Fi/config JSON and pass them directly to:

```cpp
configTzTime(timezone, NTP_SERVER1, NTP_SERVER2, NTP_SERVER3);
```

## POSIX TZ format

Every valid fixed-offset timezone string is generated by:

```text
STDoffset
```

Every valid daylight-saving timezone string is generated by:

```text
STDoffsetDST[,start[/time],end[/time]]
STDoffsetDSToffset,start[/time],end[/time]
```

Fields:

| Field | Meaning |
| --- | --- |
| `STD` | Standard-time abbreviation, 3 or more characters. Do not use digits, `+`, `-`, comma, or spaces. |
| `offset` | Value added to local time to get UTC. This means the sign is reversed from normal UTC notation: UTC+9 is `JST-9`; UTC-5 is `EST5`. |
| `DST` | Daylight-time abbreviation, 3 or more characters. |
| DST `offset` | Optional daylight offset. If omitted, daylight time is one hour ahead of standard time. |
| `start`, `end` | DST transition rules. |
| `time` | Optional transition time. If omitted, `02:00:00` is used. |

DST transition rules:

| Form | Meaning |
| --- | --- |
| `Jn` | Julian day 1-365, ignoring February 29. |
| `n` | Zero-based day 0-365, including February 29 in leap years. |
| `Mm.w.d` | Month `m`, week `w`, weekday `d`; weekday `0` is Sunday, and week `5` means the last matching weekday in the month. |

Examples:

```text
UTC0
JST-9
EST5EDT,M3.2.0,M11.1.0
CET-1CEST,M3.5.0,M10.5.0/3
```

## Notes for firmware

- Prefer a fixed-offset string only for places that do not observe DST.
- Use a DST rule for places where local civil time changes during the year.
- POSIX `TZ` strings cannot perfectly represent historical timezone changes. They describe the current recurring rule.
- If an abbreviation contains digits or signs, wrap it in angle brackets, for example `<+03>-3`.
